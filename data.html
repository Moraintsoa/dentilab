💡 Améliorations recommandées
Multi-tenant RLS
Activer Row Level Security PostgreSQL par cabinet_id. Chaque requête filtre automatiquement les données — sécurité iron-clad sans logique applicative.
Rôles extensibles
Prévoir un enum role = (docteur, assistante, secretaire, admin) dès maintenant même si seul le docteur est actif. Migration future bien plus simple.
Versioning schéma dentaire
Stocker l'historique complet des états de chaque dent via la colonne version. Ainsi on peut voir comment la santé dentaire évolue visite après visite.
Audit log
Ajouter une table audit_logs (table_name, record_id, action, old_value, new_value, user_id, at) pour traçabilité médicale réglementaire (HDS).
Notifications
Table notifications pour les rappels RDV, alertes stock inventaire, factures impayées. Avec un enum channel (email, sms, push).
Storage fichiers
Stocker les radiographies dans S3/Supabase Storage, garder uniquement l'URL signée + métadonnées en BDD. Max sécurité + performance.
Plan de traitement
Ajouter une table plans_traitement liée à patient + plusieurs actes planifiés avec statut, coût estimé, priorité. Essentiel en dentisterie.
Conformité HDS/RGPD
Chiffrement colonne (pgcrypto) pour données sensibles (numéro sécu, mutuelle). Politique de rétention données + droit à l'oubli automatisé.
